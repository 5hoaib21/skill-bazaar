# Fix Demo Payment System

## Problem
The "Continue to Payment" button on experience detail pages fails because:
1. Seed data creates host profiles with `stripeOnboardingComplete: false`
2. Backend booking creation (line 591) requires `profile.stripeOnboardingComplete === true`
3. Error thrown: "Host payment setup incomplete"

## Root Cause
`skillBazaar-server/src/seed.ts:127` sets `stripeOnboardingComplete: false` for all demo hosts.

## Solution

### 1. Update seed data to mark demo hosts as payment-ready
**File**: `skillBazaar-server/src/seed.ts`
- Change line 127: `stripeOnboardingComplete: false` → `stripeOnboardingComplete: true`
- Add mock Stripe account IDs for demo hosts
- Set `stripeChargesEnabled: true` and `stripePayoutsEnabled: true`

### 2. Verify Stripe test keys work
The `.env` files already have valid Stripe test keys:
- `sk_test_51Tg21g...` (secret key)
- `pk_test_51Tg21g...` (publishable key)

These are real Stripe test keys that support test card `4242 4242 4242 4242`.

### 3. No changes needed to Stripe checkout flow
The existing code creates Stripe Checkout sessions correctly. The only blocker was the host onboarding check.
